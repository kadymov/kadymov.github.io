📚 Краткая теория

Event Loop — как это работает

Call Stack → выполняет синхронный код.

Web APIs → setTimeout, fetch, DOM events обрабатываются браузером.

Microtask Queue (высокий приоритет): Promise.then, queueMicrotask, MutationObserver.

Macrotask Queue: setTimeout, setInterval, setImmediate (Node), I/O, UI events, MessageChannel.

Правило: после каждой макрозадачи Event Loop полностью опустошает очередь микрозадач, затем может произойти рендер, затем следующая макрозадача.

Рендеринг (Rendering Pipeline)

JS → Style → Layout → Paint → Composite

Рендер происходит между макрозадачами, обычно ~60fps (каждые 16.6ms), если браузер успевает.

requestAnimationFrame(cb) — колбэк выполнится перед следующим paint. Идеально для анимаций.

requestIdleCallback(cb) — когда браузер свободен.

Синхронный цикл блокирует UI: рендер не произойдёт, пока стек не очистится!

Порядок: sync → microtasks → render → macrotask

1. Выполнение текущей задачи (sync code).

2. Выполнение ВСЕХ микрозадач.

3. Возможный рендер (rAF → Style → Layout → Paint).

4. Одна макрозадача → goto 2.

⚠ Бесконечное добавление микрозадач заблокирует рендер!
